What is WillieOS?

WillieOS is an AI Chief of Staff named Gary Throttle who operates autonomously across email, calendar, tasks, notes, contacts, and knowledge management. Gary isn't a chatbot — he's an agent with persistent identity, cross-session memory, and the ability to execute complex workflows without hand-holding.

The Dispatch System (Crown Jewel)

This is where it gets interesting. When Gary encounters a task that requires specialized execution, the dispatch system routes it to the right agent:

Dispatch Routing:
  • Document/Report → Cowork agent (specialized document pipeline)
  • Code/Features → Claude Code (writes directly to disk, runs in project dir)
  • Research → handled inline or dispatched based on complexity

Dispatch chains allow multi-step dependency execution. Step 2 auto-triggers when Step 1 completes. Dispatch memories persist cross-session, so context isn't lost between agent calls. A handoff packet system assembles relevant memories for new dispatches.

Gary also runs on a cron scheduler with guardrails:

  • Morning autopilot (daily 7am) — email scan, calendar review, loop check, Gary's take
  • Stale loop sweep — surfaces overdue tasks
  • Pre-meeting brief — triggered before calendar events
  • Email triage — proactive inbox management
  • Nightly self-assessment — Gary grades himself on accuracy, responsiveness, proactiveness

Per-job guardrails: debounce, cooldown, daily caps, concurrency limits to prevent runaway execution.

Learning Engine

Gary learns from corrections and explicit teaching:

  • teach_gary — I explicitly teach preferences
  • record_correction — tracks when I edit/override Gary's output
  • get_gary_learnings — retrieves learned patterns by category, contact, intent

Preferences scope to global, contact-specific, context-specific, or intent-specific. Over time, Gary's drafts get eerily accurate.

These aren't criticisms — just genuine curiosity from someone who's deep in the weeds on agent orchestration and multi-system integrations. I'd love to hear how you're solving these.

The 10 Integrations

# 01
Vector Embeddings for Cognitive Shadow
Medium Semantic Search

Replace or augment the existing SQLite FTS5 full-text search in the Cognitive Shadow system with dense vector embeddings (e.g., via sqlite-vss or a local Ollama embedding model). This enables semantic recall — finding notes and loops by meaning, not just keywords.

Gary Integration

Hooks directly into shadowService.js and buildContextBlock() in aiContext.js. When Gary assembles the ~800-token context block for AI calls, semantic retrieval surfaces truly relevant notes and loops — not just lexically matching ones. Critical for the "what was I working on with [person]?" query pattern.

Use Case

You voice-capture "follow up on the pricing discussion." Gary's FTS5 finds nothing because the original note says "unit economics conversation." With embeddings, it surfaces the right loop and drafts the follow-up.

Complexity Rationale
Requires sqlite-vss Rust extension + embedding generation pipeline. No new UI needed.
Priority

# 02
AI Loop Priority Scoring
Low Predictive Analytics

Train a lightweight scoring model (or use a prompted Gemini Flash Lite call) that assigns a dynamic urgency score to Open Loops based on due date proximity, email thread activity, calendar context, and historical completion patterns from the SQLite data.

Gary Integration

Extends the existing open_loops table with a priority_score column. A background cron job (guarded by jobGuardrails.js, capped at 12/day) re-scores loops using Gemini Flash Lite. The OpenLoops widget on the dashboard renders the top-scored items first. Zero new UI work.

Use Case

You have 47 open loops. Gary quietly re-ranks them overnight: the contract review loop jumps to #1 because the counterparty sent three emails today and the meeting is tomorrow morning.

Complexity Rationale
One cron job + one DB migration. Uses existing Gemini routing, no new providers.
Priority
8.8 / 10
# 03
Contextual Email Reply Drafting
Low Natural Language Generation

Gary Integration

Extend email_triage cron job to call gmailSend.js in draft mode after classification. Uses willieProfile.js for relationship context and the existing email_bodies cache in SQLite. Haiku handles draft generation during triage — fits existing model routing.

Use Case

Investor sends a follow-up asking for updated ARR numbers. Gary drafts a reply pulling in the last known figure from willie_profile, flags it as needing your approval, and queues it in Gmail drafts before you wake up.

Complexity Rationale
gmailSend.js + draft mode API call already exist. Integration is additive to email_triage.
Priority
8.5 / 10

# 05
Relationship & Project Knowledge Graph
Medium Knowledge Management

Evolve the willie_profile table from a flat key-value store into a linked entity graph: People ↔ Projects ↔ Emails ↔ Loops ↔ Notes. Use Gemini Flash Lite to extract and link entities continuously from new data ingestion.

Gary Integration

Extends willieProfile.js and adds a willie_graph table (nodes + edges). The graph becomes the backbone for buildContextBlock() — instead of injecting raw profile text, Gary injects a structured 2-hop subgraph around today's calendar attendees. Massive context quality improvement with no token bloat.

Use Case

You ask Gary "what's the status with Acme?" It traverses the graph: Acme → contacts (3) → open loops (5, 2 overdue) → last email thread (2 days ago) → related notes (7) → surfaces a coherent summary in one response.

Complexity Rationale
DB schema design + entity extraction pipeline. New graph query logic in aiContext.js required.
Priority
8.6 / 10
# 06
PDF & Contract Intelligence
Medium Document Processing

Enable Gary to ingest PDF attachments from Gmail (contracts, reports, term sheets) via Claude's native PDF support. Extract key terms, deadlines, obligations, and action items — storing them as Open Loops with source_type='document' automatically.

Gary Integration

Extends the gmailConnector.js attachment pipeline. When a PDF arrives in a monitored label, Tauri's save_media_bytes command stores it, then a new documentProcessor.js service passes it to Claude Haiku (PDF vision). Extracted obligations land in open_loops. Cowork dispatch handles longer contracts via the existing dispatch pipeline.

Use Case

A vendor sends a 40-page MSA. Gary auto-creates three loops: "Review limitation of liability clause (section 8.3)," "Counter sign deadline: April 15," and "Legal review required: IP ownership section." All due-dated.

Complexity Rationale
Gmail attachment download + PDF parsing + structured extraction. New documentProcessor service needed.
Priority
8.0 / 10
# 07
Ambient Intelligence & Proactive Push
Medium Proactive Assistance

Move Gary from reactive (answer questions) to proactive (surface insights unprompted). A lightweight pattern-detection layer monitors email velocity, loop staleness, calendar density, and user communication patterns — pushing actionable nudges via Telegram before you think to ask.

Gary Integration

Extends garyProactiveCardService.js with a signal aggregator. Signals: 3+ emails from same sender in 24h (escalation pattern), loops older than 14 days with recent email activity (stale but relevant), calendar white-space the day before a high-stakes meeting (prep gap). Each signal triggers a telegramService.js push, gated by cooldowns in jobGuardrails.js.

Use Case

At 4pm Gary detects: tomorrow's schedule is back-to-back from 9am–6pm with no prep blocks, and two loops are overdue for attendees. It messages: "No prep time tomorrow. Block 7:30–8:30am? 2 loops need attention before the 10am."

Complexity Rationale
Signal aggregation logic is nuanced to avoid alert fatigue. Tuning phase required post-launch.
Priority

# 08
Weekly Digest & Retrospective
Low Natural Language Generation

Every Sunday evening, Gary auto-generates a rich weekly retrospective: loops closed, emails sent, meetings attended, decisions made, top people interacted with, and open items rolling into next week — formatted as a shareable briefing document pushed to Telegram and saved as a note.

Gary Integration

Piggybacks on the existing weekly_digest job already in the guardrail system. Queries open_loops (completed_at this week), email_bodies (sent), google_events (attended), transcripts (meetings). Use Case

Sunday 8pm: Gary sends "Week of March 3–9: 12 loops closed, 3 rolled over (2 overdue). 47 emails processed, 8 drafted. Top project: Acme deal (14 touchpoints). Decisions logged: 3. Next week risk: Q1 close deadline on Friday with no prep time allocated."

Complexity Rationale
weekly_digest job skeleton exists. Mostly SQL aggregation queries + Gemini prompt engineering.
Priority

# 09
Intelligent Deep Work Block Scheduling
High Scheduling Intelligence

Implement a Reclaim.ai-style scheduling engine that analyzes calendar white-space, meeting patterns, and open loop urgency to propose (and optionally auto-create) deep work blocks, buffer time, and focus sessions via the Google Calendar API — respecting working hours and meeting preferences.

Gary Integration

Requires new scheduleOptimizer.js service using Google Calendar write API (OAuth already exists in googleAuth.js). The optimizer runs as a Monday morning cron job, analyzing the week ahead using google_events data + loop priority scores from Card #02. Creates calendar events via gapi.calendar.events.insert. Claude Sonnet handles the reasoning; Haiku handles confirmation message drafting.

Use Case

Monday morning Gary analyzes the week: "You have 28 meeting hours and 4 overdue loops. I've blocked Tuesday 9–11am for the Acme proposal (your highest-priority loop) and Thursday 2–4pm as buffer before Q1 close. Confirm?" One Telegram tap creates both events.

Complexity Rationale
Calendar write API + optimization logic + confirmation UX + guardrails for auto-creation. Multi-sprint effort.
Priority

# 10
Multi-Agent Workflow Orchestration
High Workflow Automation

Extend Gary's dispatch system from single-shot agent tasks (Claude Code or Cowork) into stateful multi-step workflows where Gary orchestrates specialized sub-agents in sequence: researcher → analyst → writer → reviewer → publisher — with human-in-the-loop checkpoints via Telegram approvals.

Gary Integration

Builds on the existing dispatch.js + orchestration.js MCP tools and the resultIngestion.js pipeline. Adds a workflow_runs DB table tracking step state and output artifacts. Each step is a Cowork or Claude Code dispatch. Telegram bot handles approval gates via inline keyboard callbacks. The publish pipeline reliability issues (noted in CLAUDE.md) must be resolved as a prerequisite.

Use Case

You say "Research competitor pricing and draft a response strategy." Gary fans out: Agent 1 (Cowork) researches web; Agent 2 (Cowork) analyzes vs your pricing; Agent 3 (Claude Code) generates a structured markdown doc; Gary pings you on Telegram with "Review draft? [Yes/Edit]" before publishing to Google Drive.

Complexity Rationale
Stateful workflow engine, error recovery, partial failure handling, and approval UX. Requires publish pipeline fix first.
Priority
7.8 / 10
